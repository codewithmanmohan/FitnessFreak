QUICK START GUIDE:
================================================================================

STEP 1: SET UP MONGODB ATLAS
-----------------------------
1. Visit: https://www.mongodb.com/cloud/atlas
2. Create account → Create cluster (FREE TIER)
3. Create database user: fitnessfreakuser
4. Whitelist your IP address
5. Get connection string

STEP 2: CONFIGURE BACKEND
--------------------------
1. cd backend
2. cp .env.example .env
3. Edit .env with MongoDB URI:
   MONGODB_URI=mongodb+srv://fitnessfreakuser:PASSWORD@cluster0.mongodb.net/fitnessfreak?retryWrites=true&w=majority

STEP 3: INSTALL DEPENDENCIES
-----------------------------
1. npm install (completed ✓)

STEP 4: START SERVER
--------------------
1. npm run dev
2. Server runs on http://localhost:5000
3. Test with: curl http://localhost:5000/api/health

STEP 5: CONNECT FRONTEND
------------------------
1. Update frontend to call http://localhost:5000/api/*
2. Store JWT token from login in localStorage
3. Send token in Authorization header for protected routes

AUTHENTICATION FLOW:
================================================================================

1. User Signup
   POST /api/auth/signup
   → User created, password hashed, JWT token returned

2. User Login
   POST /api/auth/login
   → Credentials verified, JWT token returned

3. Protected Requests
   GET /api/auth/profile
   Headers: Authorization: Bearer {token}
   → User verified via JWT, data returned

4. Token Expiry
   Token valid for 7 days
   → User needs to login again after expiry

5. Authorization
   User: Can access own data
   Coach: Can manage own profile
   Admin: Can manage supplements & approvals

TROUBLESHOOTING:
================================================================================

"Cannot connect to MongoDB"
  → Check MONGODB_URI in .env
  → Verify IP is whitelisted
  → Check database user password

"Port 5000 already in use"
  → Change PORT in .env
  → Or kill process on port 5000

"CORS error from frontend"
  → Check CORS_ORIGIN in .env
  → Should be http://localhost:5173

"Module not found"
  → Run npm install in backend directory

"Invalid token"
  → Check JWT_SECRET in .env
  → Token may have expired

See SETUP.md for more troubleshooting.
